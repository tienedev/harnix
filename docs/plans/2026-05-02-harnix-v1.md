# harnix v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a standalone Nix flake exposing a home-manager module that declaratively configures MCP servers (Claude Code + Pi), Claude Code plugins, and agent skills from a pinned community registry.

**Architecture:** A single `homeManagerModules.default` output imports `options.nix` (option schema) + three renderers (`claude-code.nix`, `pi.nix`, `skills.nix`) that read `config.ai.*` and write the appropriate files. A `registry.nix` maps `"name@marketplace"` strings to pinned `fetchFromGitHub` derivations; consumers can override any pin via `ai.plugins.claude-code.sources`.

**Tech Stack:** Nix flakes, home-manager module system, `builtins.toJSON`, `pkgs.runCommand`, `pkgs.fetchFromGitHub`

---

## File map

| File | Responsibility |
|------|---------------|
| `flake.nix` | Inputs, outputs (`homeManagerModules.default`, `checks`) |
| `registry.nix` | Pinned fetchFromGitHub sources for all known plugins |
| `lib.nix` | Pure helpers: `mkPluginDrv`, MCP entry formatters |
| `modules/options.nix` | Option types for `ai.mcpServers`, `ai.plugins.claude-code` |
| `modules/default.nix` | Imports all sub-modules |
| `modules/renderers/pi.nix` | Writes `~/.pi/agent/mcp.json` |
| `modules/renderers/claude-code.nix` | Writes `~/.claude.json`, `installed_plugins.json`, plugin symlinks |
| `modules/renderers/skills.nix` | Writes `~/.agents/skills/<plugin>/` from plugins with `hasSkills = true` |
| `checks/default.nix` | Eval + content tests for `nix flake check` |

## Task 3: lib.nix — helper functions

**Files:**
- Modify: `lib.nix`

- [ ] **Step 1: Write `lib.nix`**

```nix
{ lib, pkgs }:
{
  # Extract a subpath from a source into a standalone derivation.
  # Used to isolate a single plugin from a marketplace mono-repo.
  mkPluginDrv = { src, subpath, name }:
    pkgs.runCommand "harnix-plugin-${name}" {} ''
      cp -r ${src}/${subpath} $out
    '';

  # Extract the skills/ subdir from a plugin derivation.
  mkSkillsDrv = { src, subpath, name }:
    pkgs.runCommand "harnix-skills-${name}" {} ''
      if [ -d ${src}/${subpath}/skills ]; then
        cp -r ${src}/${subpath}/skills $out
      else
        mkdir $out
      fi
    '';

  # Format one mcpServer entry for Claude Code's ~/.claude.json
  mkClaudeCodeMcpEntry = _name: server:
    if server.type == "stdio" then
      { inherit (server) type command args; }
    else if server.type == "http" then
      { inherit (server) type url; }
      // lib.optionalAttrs (server.headers != {}) { inherit (server) headers; }
    else  # sse
      { inherit (server) type url; };

  # Format one mcpServer entry for Pi's ~/.pi/agent/mcp.json
  mkPiMcpEntry = _name: server: {
    url       = server.url;
    lifecycle = "lazy";
  };
}
```

```nix
{ config, lib, pkgs, ... }:
let
  cfg      = config.ai;
  harnixLib = import ../../lib.nix { inherit lib pkgs; };
  registry  = import ../../registry.nix { inherit pkgs; };

  # Resolve a plugin id: override if provided, else registry entry
  resolveEntry = id:
    let
      override = cfg.plugins.claude-code.sources.${id} or null;
      entry    = registry.claude-code.${id}
        or (throw "harnix: unknown plugin '${id}' — add it to registry.nix or set ai.plugins.claude-code.sources.\"${id}\"");
    in
    if override != null then entry // { src = override; } else entry;

  # Build a derivation containing only the plugin directory
  resolvedPlugins = lib.genAttrs
    (cfg.plugins.claude-code.user
      ++ lib.concatLists (lib.attrValues cfg.plugins.claude-code.project))
    (id:
      let e = resolveEntry id;
      in e // { drv = harnixLib.mkPluginDrv { inherit (e) src subpath; name = "${e.marketplace}-${e.plugin}"; }; }
    );

  # ~/.claude.json — only mcpServers for now; other keys (numStartups etc.) are runtime state
  claudeJson = builtins.toJSON {
    mcpServers = lib.mapAttrs harnixLib.mkClaudeCodeMcpEntry cfg.mcpServers;
  };

  # installed_plugins.json
  mkEntry = scope: id: extraAttrs:
    let p = resolvedPlugins.${id};
    in {
      inherit scope;
      installPath = "${config.home.homeDirectory}/.claude/plugins/cache/${p.marketplace}/${p.plugin}/${p.version}";
      version     = p.version;
      installedAt = "1970-01-01T00:00:00.000Z";
      lastUpdated = "1970-01-01T00:00:00.000Z";
      gitCommitSha = p.src.rev or "unknown";
    } // extraAttrs;

  installedPluginsJson = builtins.toJSON {
    version = 2;
    plugins =
      # User-scope entries
      lib.listToAttrs (map (id: {
        name  = id;
        value = [ (mkEntry "user" id {}) ];
      }) cfg.plugins.claude-code.user)
      # Project-scope entries (merged)
      // lib.foldl (acc: pair:
        let
          projectPath = pair.name;
          ids         = pair.value;
        in
        acc // lib.listToAttrs (map (id: {
          name  = id;
          value = [ (mkEntry "local" id { inherit projectPath; }) ];
        }) ids)
      ) {} (lib.attrsToList cfg.plugins.claude-code.project);
  };

  # home.file entries for plugin dirs (symlinks to Nix store)
  pluginDirEntries = lib.listToAttrs (map (id:
    let p = resolvedPlugins.${id};
    in {
      name  = ".claude/plugins/cache/${p.marketplace}/${p.plugin}/${p.version}";
      value = { source = p.drv; };
    }
  ) (lib.attrNames resolvedPlugins));

  hasPlugins = cfg.plugins.claude-code.user != []
            || cfg.plugins.claude-code.project != {};
in
{
  home.file = lib.mkMerge [
    { ".claude.json".text = claudeJson; }

    (lib.mkIf hasPlugins (
      { ".claude/plugins/installed_plugins.json".text = installedPluginsJson; }
      // pluginDirEntries
    ))
  ];
}
```

Skills from plugins with `hasSkills = true` are deployed to `~/.agents/skills/<plugin-name>/`, which Pi scans natively. Each `~/.agents/skills/<plugin>/` directory contains the individual skill subdirectories (e.g. `brainstorming/`, `debugging/`).

- [ ] **Step 1: Write `modules/renderers/skills.nix`**

```nix
{ config, lib, pkgs, ... }:
let
  cfg       = config.ai;
  harnixLib = import ../../lib.nix { inherit lib pkgs; };
  registry  = import ../../registry.nix { inherit pkgs; };

  resolveEntry = id:
    let
      override = cfg.plugins.claude-code.sources.${id} or null;
      entry    = registry.claude-code.${id}
        or (throw "harnix: unknown plugin '${id}'");
    in
    if override != null then entry // { src = override; } else entry;

  allPluginIds = lib.unique (
    cfg.plugins.claude-code.user
    ++ lib.concatLists (lib.attrValues cfg.plugins.claude-code.project)
  );

  # Plugins that declare skills
  skillPlugins = builtins.filter (id: (resolveEntry id).hasSkills or false) allPluginIds;

  # For each skill plugin: a derivation that contains only the skills/ subdir contents.
  # ~/.agents/skills/<plugin>/ → Nix store dir with brainstorming/, debugging/, etc.
  skillDirEntries = lib.listToAttrs (map (id:
    let e = resolveEntry id;
    in {
      name  = ".agents/skills/${e.plugin}";
      value = {
        source = harnixLib.mkSkillsDrv {
          inherit (e) src subpath;
          name = "${e.marketplace}-${e.plugin}";
        };
      };
    }
  ) skillPlugins);
in
lib.mkIf (skillPlugins != []) {
  home.file = skillDirEntries;
}
```

# harnix

> Declarative AI agent configuration for NixOS / home-manager.

One Nix declaration drives all your AI tools. MCP servers, Claude Code plugins, and agent skills are pinned, reproducible, and overwritten on rebuild. CLI additions don't persist — the config is the truth.

## Supported tools

- **Claude Code** — `~/.claude.json` (mcpServers), `~/.claude/plugins/` (plugins + installed_plugins.json)
- **Pi** — `~/.pi/agent/mcp.json`
- **Both** — `~/.agents/skills/` (Agent Skills standard, discovered natively by Pi)

## Usage

```nix
# flake.nix
inputs.harnix.url = "github:tienedev/harnix";

# home.nix
{ inputs, ... }: {
  imports = [ inputs.harnix.homeManagerModules.default ];

  ai.mcpServers.titi-browser = {
    type = "sse";
    url  = "http://titi-browser:9223/sse";
  };

  ai.plugins.claude-code.user = [
    "superpowers@claude-plugins-official"
    "obsidian@obsidian-skills"
  ];

  # Project-scope plugins
  ai.plugins.claude-code.project."/home/titi/Projets/pentest" = [
    "burpsuite-project-parser@trailofbits"
  ];

  # Override a registry pin (fork or custom commit)
  ai.plugins.claude-code.sources."superpowers@claude-plugins-official" =
    pkgs.fetchFromGitHub {
      owner = "anthropics"; repo = "claude-plugins-official";
      rev = "my-commit"; hash = "sha256-...";
    };
}
```

## ⚠️ Runtime state

`~/.claude.json` and `~/.pi/agent/mcp.json` are **fully owned by harnix**. Any changes made via CLI (`claude mcp add`, etc.) are overwritten on the next rebuild. To persist a change, declare it in your Nix config.

## Registry

`registry.nix` maps `"name@marketplace"` identifiers to pinned `fetchFromGitHub` calls. Run `nix flake update` to bump all pins. Submit a PR to add or update entries.

## Contributing

PRs welcome for new registry entries. Each entry needs: `marketplace`, `plugin`, `version`, `src` (fetchFromGitHub with exact `rev` + `hash`), `subpath`, `hasSkills`.
